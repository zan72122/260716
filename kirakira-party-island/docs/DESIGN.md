# 設計メモ — きらきら☆パーティーアイランド

## ねらい

- 対象は 4 歳児。**文字が読めなくても遊べる**ことを最優先に、
  アイコン・色・音・アニメーションで情報を伝える。
- 「スーパー マリオパーティ ジャンボリー」のような
  *すごろく+ミニゲーム* の構造と、Switch らしい明るいトゥーン 3D の質感を、
  ブラウザ(iPhone/iPad Safari)だけで再現する。
- ネガティブな体験を作らない:マイナスのマスなし、ミニゲームで負けても
  参加コインがもらえ、最後は全員でお祝い。

```
js/
  config.js           調整値・キャラ定義・マス配置・URLパラメータ
  main.js             レンダラ・カメラリグ・入力・メインループ
  world.js            島・海・空・雲・装飾・すごろくタイル
  characters.js       どうぶつフレンズ4体(プリミティブ組み立て+アニメ)
  board.js            ターン制すごろく進行(async/await の状態遷移)
  ui.js               HUD/バナー/タイマー等の DOM 操作
  audio.js            WebAudio 合成の SFX・BGM
  gfx.js              トゥーン素材・Canvas テクスチャ・シェーダー
  particles.js        紙ふぶき・キラキラ・コイン・土ぼこり
  minigames/
    harness.js        ミニゲーム進行係(説明→321→プレイ→結果)
    base.js           共通基盤(専用シーン・CPU得点・レイキャスト)
    balloon.js fruits.js race.js stars.js
```

## 見た目(Switch 風トゥーンの作り方)

- **セル調シェーディング**: `MeshToonMaterial` に 4 段の
  `NearestFilter` グラデーションマップ。色はパステル基調のパレットで統一。
- **輪郭線**: キャラの主要パーツに反転ハル(BackSide・1.045 倍)の黒メッシュ。
- **ライティング**: HemisphereLight(空色/草色)+ 影付き DirectionalLight。
  `PCFSoftShadowMap`、`ACESFilmicToneMapping`、sRGB 出力。
- **島**: 1 枚の PlaneGeometry を高さ関数 `islandHeight(x,z)` で変位し、
  高さで 砂浜→草→丘 の頂点カラーを塗る。タイル・木・キャラの接地も同じ関数を参照。
- **海**: 頂点をゆらす自作 ShaderMaterial。2 トーン+同心円ストライプで
  アニメ調の水面。島のまわりに白い波リング。
- **空**: グラデーションドーム+加算グローの太陽+ゆっくり流れる雲。
- **にぎやかし**: 中央のスタータワー、虹、気球、ちょうちょ、旗、お花。

## ゲームフロー

```
タイトル(キャラ選択)
 └→ ボード: ラウンド開始 → 4人が順にサイコロ→移動→マスイベント
      └→ ラウンド末にミニゲーム(勝者+5コイン、参加+1)
 └→ 5ラウンド後: 結果発表(スター→コイン順)→ 王冠+紙ふぶき+ダンス
 └→ タップで最初から
```

- 進行は `board.js` の async 関数群。アニメは自前の Tween キューで
  フレーム駆動し、`await` で直列に流す。
- プレイヤー以外の 3 人は CPU(自動でサイコロ)。
- スターは取られるたび、離れたマスへ「おひっこし」。

## 4歳児向け UI の指針

- タッチターゲットは最小でも 64px 相当。サイコロボタンは画面の 1/4 幅。
- ミニゲームの操作は「タップ」か「よこにドラッグ」のみ。長押し・スワイプ・
  マルチタッチは使わない。
- 失敗ペナルティなし(風船の空振り、フルーツ落下もマイナスにしない)。
- CPU の得点はプレイヤーがやや勝ちやすいレートで乱数シミュレーション。
- 文字はすべて ひらがな+絵文字。バナーは大きく短く。

## パフォーマンス

- `devicePixelRatio` は 2 でクランプ。ジオメトリは低ポリ、素材は色ごとに共有キャッシュ。
- お花はインスタンシング、紙ふぶきは `InstancedMesh` プール(260枚)。
- ミニゲームは専用シーンを都度生成し、終了時にジオメトリを dispose
  (借りてきたキャラは先に島へ返してから)。

## テスト

Playwright + ヘッドレス Chromium(SwiftShader)で
タイトル→ボード→各ミニゲーム→結果発表まで自動プレイし、
コンソールエラーゼロと画面キャプチャを確認済み。
ソフトウェアレンダリングでは実時間より遅く進むため、
検証時は `?rounds=1&mgtime=8&mg=…` で短縮する。
