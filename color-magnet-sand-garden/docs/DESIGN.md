# 設計メモ — 仕様と実装の対応

元仕様「カラーじしゃく砂もよう庭」の各要件を、どこでどう実装したかの対応表。

## 「磁石感」の6つの強化(第2次改善)

| アイデア | 実装 |
| --- | --- |
| 磁力の急坂(じわじわ→一気) | 場の強さ `s = 0.12u + 0.88u⁴`(u=1-d/R)。`snapRadius` 内は確率を待たず即捕獲+初速バースト。捕獲ばねは磁石に近いほど硬い(`nearSpringBoost`)。遠くの立ち粒は `sin` でそよぐ。 |
| 力線の見える化 | 寝粒の向きを2磁石の**ベクトル合成場**に揃える(2磁石間で力線が曲がって繋がる)。可視化半径は捕獲半径の1.6倍。磁石からの角度で8本の「筋」(スポーク)に乗った粒だけ長く明るい。 |
| 砂の連鎖吸い | 実効捕獲半径 = `max(fieldRadius, 山の半径+40)` で、大きな山は縁の接触で寝砂を取り込む。捕獲の瞬間の位置を毎フレーム最大12件記録し、半径20px の寝粒へ磁石方向の微小インパルス(磁石→砂→砂の伝播)。 |
| くっつき持ち運び | ばね強さを山の芯ほど硬く(`coreTighten`)=山ごと運ぶ感触。平滑化速度(`dragSpeed`)が 400px/s を超えた状態で指を離すと、外周(slotR>0.55×山半径)がちぎれて置き跡になる。 |
| 橋の主役化 | 参加しきい値が距離で動く(近い=大勢で太い、遠い=精鋭の糸)。`bridgeOn` ヒステリシス(>0.35 で ON、≤0.02 で OFF)。ON でグリッサンド、OFF で「ぱちんっ」+伸びていた粒が自分の磁石へ反動で縮む。橋参加中の粒はちぎれ判定を免除。 |
| ふりふり=磁力を切る | 3幕構成: (1)全捕獲を一斉解放して山がふわっと崩れ、磁石は目を閉じてくすみ、光輪が消える(`fieldOff`)+下降音。(2)色の波が端から走って平らに(既存)。(3)波の後 0.4 秒で磁力が再点灯(pop 音+wobble)し、砂がじわっと戻り始める。 |

## コンセプト

> 画面下の磁石を動かすと、色のついた磁性砂が遅れて集まり、山、尾、渦、橋、マーブル模様を作る。

| 要素 | 実装 |
| --- | --- |
| 遅れて集まる | 捕獲は確率的(場が強いほど早く捕まる)+ ばね追従に粒ごとの強さゆらぎ(`kJit`)。`sim.js` の捕獲判定と CAPTURED 更新。 |
| 山 | 捕獲した粒に砂山内の「席」(半径・角度)を割り当てる。席は捕獲順に外側へ広がる(`_capture`)。 |
| 尾 | ばねの遅れで磁石の後ろに粒が引きずられる。目標から `releaseDist` 以上離れると解放され、その場に落ちて軌跡になる。 |
| 渦 | 磁石の進行方向の変化(円運動)が砂山の席フレームを回転させる(`slotRot` / `swirlGain`)。 |
| 橋 | 2磁石間の距離から橋係数を計算し、相手方向の席の粒ほど相手側へ引き伸ばす。両方の磁石で対称に行うため、橋は両側から育つ。 |
| マーブル | 粒は `colorIdx` を一生保持し、色を混算しない。捕獲順の年輪 + 引きずりで模様ができる。 |

> 砂は弱い場所へ置かれるとその場に残り、シェイクした時だけ平らに戻る。

- 解放された粒は摩擦で減速し、`restSpeed` 未満で SETTLED として静止。以後は磁石が近づくまで動かない。
- 平らに戻すのは「ふりふり」ボタンのみ(SMOOTHING 状態で盤面全体へ一様に再配置)。

## タッチ操作

| 仕様 | 実装 |
| --- | --- |
| 磁石の1指ドラッグ | `input.js`: `grabRadius` 内から引くと直接ドラッグ。遠くから引いても近い磁石が追いかける(4歳向けの許し)。 |
| 空所のタップ → 2個目の磁石 / 移動先 | `sim.tapAt()`: 2個目がなければ配置、あれば近い磁石が `glideSpeed` で滑走(ゆっくり動くので砂を引き連れる)。 |
| 大きなシェイクボタン | 92px の「ふりふり」ボタン。`startShake()` が色の波を起動。 |
| 磁石は指より大きく、入力点より上へ表示 | スプライトは約96px。`dragMagnet()` が掴んだ瞬間のオフセットから `liftOffset`(78px 上)へ 0.22 秒で移行。 |
| 「極」や反発の説明は入れない | すべての磁石は引き寄せのみ。UI に極の表記なし。2個目は色だけ違う(赤/青)。 |

## 非言語フィードバック

| 仕様 | 実装 |
| --- | --- |
| 入力前から近くの粒を数個立たせる | SETTLED 粒は場の強さ×個体差(`standJit`)が閾値を超えると磁石方向を向いて伸びる。個体差により「数個だけ」立つ。 |
| 場が強いほど粒を長く明るく | `len` と `bright`(3段階の明度バケツ)を場の強さ・速度から毎フレーム計算。 |
| 粒単位の色保持(マーブル) | 描画は色×明度ごとのパスに分けてバッチ化し、色の混算はしない(`render.js`)。 |
| 速く動かすと高いさらさら音 / 落ち着くと低いふわっ | `audio.js`: バンドパスノイズのカットオフと音量を磁石速度に追従。捕獲粒の平均速度が高→低へ落ちた瞬間に低いノイズの息(`settlePuff`)。 |
| 橋は両側から粒が伸びる | 両磁石の粒が対称に相手側へ伸びる(上記)。 |
| リセットは色の波が端から走る | 波前線は「盤面の縁からの距離」で定義。通過した粒から順に光って新しい場所へ滑る。画面も 0.5 秒揺れる。 |

## 4歳児向けの配慮

- 文字に頼らない: ボタンは形・色・アイコン中心。タイトルも文字なし(磁石+タップを促す指のアニメ、どこをタップしても開始)。
- タッチ即応: pointerdown と同フレームで近くの寝粒が跳ね、白いリングが出て、さらっと音が鳴る(`pokeAt`)。入力→視覚変化 100ms 未満。
- おさそい: 12秒無操作で磁石がぷるっと震え、近くの粒が数個跳ねる(以降8秒ごと、触れば即中断)。
- 誤操作に強い: どこをドラッグしても遊べる / ピンチ・ダブルタップ拡大・スクロールを抑止 / 3本以上の同時タッチ(手のひら)はタップ扱いしない。
- 2個目の磁石は上からぽとっと落ちて着地する(突然現れない)。落下中は磁場・合体が無効。
- 磁石に顔を付けて「道具」ではなく「ともだち」に。磁力オフ中は目を閉じて眠る。
- 音はタイトルのタップでアンロック(iOS の自動再生制限対応)。ミュート永続化。

## 画面回転(原則: 回転で遊びを失わない)

- リサイズ・回転時は**トレイ矩形基準の正規化座標**で全永続状態(粒・目標・磁石・滑走先)を
  アフィン再配置する(`Sim.resize`)。描いた模様の相対配置は完全に維持される。
- 砂山の席(slotR/slotA)は磁石ローカルなので変換せず、山は丸いまま再形成される。
- 描画側の演出座標(波紋・タッチリング)も同じ係数で追従(`Renderer.remap`)。

## 性能

- 粒は並列 TypedArray(GC なし)。描画は最大 18 パス(6色×3明度)。
- 実測 FPS で粒数を 1,800〜5,600 に自動調整(`main.js tunePerformance`)。
- devicePixelRatio は 2 で上限。盤面(木枠)はオフスクリーンに事前描画。
- 追加コストは軽量: 力線の atan2 は磁石近傍の立ち粒のみ、連鎖ツグは毎フレーム最大12件、
  pokeAt は pointerdown 時のみの全粒走査。

## 確認済み事項(Playwright / Chromium、20項目自動テスト)

- iPhone 縦(390×844)・iPad 横(1180×820)でコンソールエラーなし。
- タイトルは画面のどこをタップしても開始する。
- touchStart から 90ms 以内に半径90px の粒が跳ねる(タッチ即応)。
- snapRadius 内の粒は1フレームで捕獲される(急坂)。
- 静止磁石の周囲 100〜200px の寝粒の向きが磁石方向と平均 |cos|>0.9(力線)。
- 高速リリースで山の外周がちぎれる(575→259粒)。
- 橋: 接近で "bridge" が1回、離間で "bridgeBreak" が1回だけ発火(ヒステリシス)。
- ふりふり: 直後に捕獲0+磁場オフ → 波の間は捕獲0のまま → reboot 後に再捕獲。
- 3点同時タッチでは磁石2が誤配置されない。
- 回転(390×844→844×390)後の正規化座標の平均誤差 0.0006 < 0.01(模様維持)。
